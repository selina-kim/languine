- Google OAuth:  
  1. **Frontend \- Normal behaviour:**   
    1. User authenticates via Google OAuth by clicking sign-in button, gets redirected to Google consent screen,   
    2. returns with token, and app stores JWT for authenticated requests.   
  2. **Backend \- Normal behaviour:** The Authentication component acts as a boundary between external identity providers and internal application logic.   
    1. It receives ID tokens from the frontend,   
    2. verifies them using Google OAuth services,  
    3. maps verified identities to internal user records.   
    4. The component generates and returns application-specific JWTs that enable stateless authentication for future requests. It also validates JWTs on protected endpoints and supports token refreshing.
